crud (string)

only D3W

(C)reate, (R)ead, (U)update, (D)elete on this EO.

This uses a "byte" pattern. Each ordinal position corresponds with the action of Create, Read, Update and Delete.

0 == Action NOT Allowed. 1 == Action Allowed.

Example: 0110 would be Cannot Create, Allow Read, Allow Update, Cannot Delete.

Example: 1111 would be Allow Create, Allow Read, Allow Update, Allow Delete.

In the case of removing an EO from a relationship, if the parent EO relationship is mandatory it will show no delete button. You need to provide an additional rule for "isEntityDeletable"
